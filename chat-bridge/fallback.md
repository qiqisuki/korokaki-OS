# 降级与容错策略

> 外部依赖挂了浮窗不能崩——优雅降级，自动恢复。

## 错误分类

| 级别 | 场景 | 行为 |
|---|---|---|
| 🟡 轻微 | API 单次超时 | 气泡显示 "姐姐在想..."，3 秒后自动重试 |
| 🟠 中等 | Claude API 持续不可用（>30s） | 气泡显示 "信号不太好……等等" + 托盘通知 "连接中断" |
| 🔴 严重 | API 完全不可用（>5min） | 角色切换到 sleep 表情，气泡提示离线。后台持续重连 |
| ⚫ 灾难 | 网易云 API 挂掉 | 自动切本地播放模式（assets/music/ 文件夹），托盘提示 |
| ⚫ 灾难 | 微信桥接断开 | 托盘图标变灰，每 10 秒自动重连 wechat-mcp |

## 通用降级原则

1. **用户可见但不打扰** — 出问题显示在气泡里，不弹系统弹窗
2. **自动恢复优先** — 所有外部依赖有 reconnect 逻辑
3. **离线不崩溃** — 浮窗本体是纯本地渲染，没网络照样眨眼
4. **状态持久化** — 断线前最后状态保存，恢复后继续

## 心跳检查

每 30 秒 ping 一次 Claude API（轻量 `GET /v1/models` 或类似 endpoints）。

## 实现位置

- `chat-bridge/heartbeat.js` — 心跳 + 重连
- `chat-bridge/fallback.js` — 降级消息池（随机选一条兜底回复）
- `main/health.js` — 主进程健康监控面板
